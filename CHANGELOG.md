# Changelog

Все изменения в проекте SwapIP документируются в этом файле.

Формат основан на [Keep a Changelog](https://keepachangelog.com/ru/1.0.0/),
и проект придерживается [Семантического версионирования](https://semver.org/lang/ru/).

## [v1.0.2] - 2026-04-12

### Добавлено
- **Система обновления через GitHub Releases**: полный набор скриптов для автоматического обновления компонентов sender и recepient.
- **Скрипт подготовки релиза** (`scripts/prepare-release.sh`): автоматически создаёт структуру файлов для релиза на основе Git-тега.
- **Общие функции** (`scripts/common-functions.sh`): кроссплатформенные утилиты для логирования, проверки зависимостей, работы с файлами.
- **GitHub API клиент** (`scripts/github-api.sh`): взаимодействие с GitHub Releases API (получение информации о релизах, проверка rate limit).
- **Скрипты обновления**:
  - `update-sender.sh`, `update-recepient.sh` – автоматическое обновление до последней версии (также выполняет первичную установку если компонент не установлен).
  - `rollback-sender.sh`, `rollback-recepient.sh` – откат к предыдущей версии в случае проблем.
- **Systemd службы** – для автоматического запуска компонентов и периодической проверки обновлений:
  - `swapip-sender.service`, `swapip-recepient.service` – службы для компонентов.
  - `swapip-check-update.service`, `swapip-check-update.timer` – таймер для автоматической проверки обновлений.
- **Интеграция с Makefile**: новые цели `prepare-release`, `install-scripts`, `test-update-system`, `test-full-cycle`.
- **Документация** в README.md: подробное описание системы обновления, архитектуры, инструкций по использованию.

### Изменено
- **Makefile**: переработана цель `all` (теперь только `build-l`), добавлены цели для системы обновления.
- **README.md**: расширен раздел "Система обновления через GitHub Releases" с полным описанием.

### Исправлено
- Нет критических исправлений в этом релизе.

### Известные проблемы
- На Windows отсутствует поддержка systemd, функциональность скриптов обновления ограничена.

## [v1.0.0] - 2026-04-12

### Добавлено
- **Базовые компоненты**:
  - `sender`: утилита для определения внешнего IP и отправки на recepient.
  - `recepient`: HTTP-сервер для приёма IP и обновления конфигов nginx.
- **Конфигурация через переменные окружения**: поддержка `.env` файлов.
- **Логирование**: на основе zap, с возможностью вывода в консоль или файл.
- **Безопасность**: Basic Auth для защиты HTTP-эндпоинта.
- **Документация**: README.md с описанием архитектуры, конфигурации, примеров использования.
- **Makefile**: цели для сборки под Linux и Windows.

### Изменено
- Нет (первый релиз).

### Исправлено
- Нет.
