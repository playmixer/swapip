# Синхронизация ip адреса на сервере

- Sender - получение внешнего адреса и отправка в Recepient
- Recepient - слушает порт 8080 и обновляет файлы конфига, потом выполняет скрипт с перезапуском Nginx

## Конфигурация

Переменные окружения:

- `RECEPIENT_NGINX_CONF` — путь к файлу со списком конфигурационных файлов nginx (каждый путь на новой строке). Ранее использовался список через точку с запятой.
- `RECEPIENT_SCRIPT` — путь к скрипту перезапуска nginx.
- `RECEPIENT_ADDRESS` — адрес и порт, на котором слушает Recepient (по умолчанию `:8080`).
- `SENDER_REMOTE_ADDRESS` — URL адреса Recepient для отправки IP.
- `SENDER_BASIC_AUTH` — базовая аутентификация для отправки.
- `STORAGE_IP` — путь к файлу хранения текущего IP (по умолчанию `ip.data`).
- `LOG_LEVEL` — уровень логирования.
- `LOG_PATH` — путь к файлу логов.

## Формат файла списка конфигураций nginx

Файл должен содержать пути к конфигурационным файлам nginx, по одному на каждой строке. Пустые строки и строки, начинающиеся с `#`, игнорируются.

Пример:
```
/etc/nginx/nginx.conf
/etc/nginx/sites-enabled/default
# /etc/nginx/extra.conf
```
